# Intelligent Healthcare Diagnostic Assistant

**Capstone Project — Introduction to AI (13-Week Course)**

An end-to-end AI system that combines an intelligent agent, first-order-logic
inference, Bayesian probabilistic reasoning, classical machine learning, a
deep neural network, fuzzy logic, and STRIPS-style automated planning into
one unified healthcare diagnostic and treatment-recommendation platform.

## How the system fits together

```
Patient Input (symptoms, vitals, age, history)
        │
        ▼
Module 1: Intelligent Agent  (perceive → think → act)
        │
        ├──▶ Module 2: Knowledge Base       (FOL, forward/backward chaining)
        ├──▶ Module 3: Bayesian Network     (Naive Bayes posterior probability)
        ├──▶ Module 4: ML Classifier        (Decision Tree / Random Forest / GBM)
        ├──▶ Module 5: Neural Network       (Keras MLP, 128→64→32→8)
        └──▶ Module 6: Fuzzy Severity       (fuzzification → rules → defuzzification)
        │
        ▼
Module 7: AI Treatment Planner  (STRIPS + BFS state-space search)
        │
        ▼
Final Report: Diagnosis, Confidence, Urgency, Treatment Plan
```

No single module gives the final answer alone — the Agent combines every
module's diagnosis and confidence score into one recommendation, then hands
that off to the planner to generate a step-by-step treatment plan.

## Project structure

```
├── data/
│   ├── symptoms.csv          # symptom reference table
│   ├── diseases.csv          # disease reference table
│   └── sample_patients.csv   # 8 hand-crafted test patients w/ expected diagnosis
├── modules/
│   ├── agent.py              # Module 1 — Intelligent Agent (PEAS, perceive/think/act)
│   ├── knowledge_base.py     # Module 2 — FOL + forward/backward chaining
│   ├── bayesian_net.py       # Module 3 — Naive Bayes probabilistic reasoning
│   ├── ml_classifier.py      # Module 4 — Decision Tree / Random Forest / Gradient Boosting
│   ├── neural_network.py     # Module 5 — Keras deep neural network
│   ├── fuzzy_controller.py   # Module 6 — Fuzzy severity scoring
│   └── planner.py            # Module 7 — STRIPS treatment planner (BFS)
├── evaluation/
│   ├── metrics.py            # accuracy / precision / recall / F1 / confusion matrix
│   ├── visualizations.py     # confusion-matrix heatmaps + module comparison chart
│   └── evaluate_system.py    # full evaluation pipeline — run this for all metrics/plots
├── reports/                  # generated confusion matrices, comparison chart, final report
├── app.py                    # main application entry point
├── requirements.txt
└── README.md
```



## Testing an individual module

Every module can be tested standalone before being wired into `app.py`:

```bash
python3 -m modules.agent
python3 -m modules.knowledge_base
python3 -m modules.bayesian_net
python3 -m modules.ml_classifier
python3 -m modules.neural_network
python3 -m modules.fuzzy_controller
python3 -m modules.planner
```

## Results summary

See `reports/module_comparison.png` and `reports/confusion_matrix_*.png`
(generated by `evaluation/evaluate_system.py`) for the full accuracy /
precision / recall / F1 breakdown across the Knowledge Base, Bayesian
Network, and ML Classifier on a 120-patient labeled synthetic test set
spanning all 8 diseases. In short: the rule-based Knowledge Base is the
weakest (rules only fire on exact symptom matches), the Bayesian Network
does meaningfully better by reasoning under uncertainty, and the ML
Classifier (Gradient Boosting) is the strongest, at ~92% accuracy.

## Known limitations / corrections made while building this

- **Planner bug (fixed):** the original action library had no route from
  any non-COVID diagnosis to a confirmed diagnosis, and no treatment path
  at all for cardiac events or diabetes. Added a generic lab-confirmation
  action plus disease-specific treatment actions — see comments in
  `modules/planner.py`.
- **Missing imports (fixed):** `fuzzy_controller.py`, `ml_classifier.py`,
  and `neural_network.py` referenced `typing.Dict`/`List` in type hints
  without importing them, which would crash on import.
- **Inconsistent disease vocabulary:** `bayesian_net.py` and
  `knowledge_base.py` use different disease-label spellings/suffixes than
  `ml_classifier.py`/`neural_network.py` (e.g. `"cardiac"` vs.
  `"cardiac_event"` vs. `"cardiac_event_suspected"`). `app.py` normalizes
  these before handing a diagnosis to the planner, but this means the
  agent's majority vote across modules under-counts true agreement — a
  good next improvement would be a single shared disease-label enum used
  by every module.
- **`requirements.txt`** in this build only lists packages the code
  actually imports (numpy, pandas, matplotlib, seaborn, scikit-learn,
  tensorflow, scipy). Packages like `pgmpy`, `scikit-fuzzy`, `nltk`,
  `networkx`, and `gymnasium` appeared in the original course template but
  aren't used anywhere in this codebase — the Bayesian and fuzzy-logic
  modules are hand-implemented with numpy instead.
